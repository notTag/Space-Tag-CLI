# [004] Active blink can stale-paint pill focused-blue after focus change

- Status: open
- Created: 2026-06-01
- Severity: low
- Bump: patch

## Repro
1. Be ON the agent's window when its turn ends (the active-blink path in flash-listener.sh, `ON_TRIGGER=1`).
2. While the blink loop is still running (~N*0.7s ≈ 3.5s), switch focus away to a different space/window so that pill becomes unfocused.

## Expected
Once focus leaves, the pill settles on its unfocused steady color — space_change already repainted it correctly.

## Actual
The blink loop captures `REVERT_BG="$COLOR_PILL_BG_FOCUSED"` once (flash-listener.sh:98) and keeps writing it for the rest of the loop. Its final revert overwrites the unfocused paint, leaving the pill focused-blue while it is no longer focused. The window-destroy reconciler does not help: active blinks create no pending marker, so nothing reconciles this path.

## Notes
Fix options:
- Re-query the focused space before each revert and pick BG accordingly, or
- Trigger `space_change` after the blink loop so space.sh does the authoritative final paint (handles focus + any markers). Preferred — space.sh is the single source of truth for steady-state color.
