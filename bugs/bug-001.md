# [001] Stale pill color when flashed/solid-hold app closes

- Status: Resolved
- Created: 2026-06-01
- Resolved: 2026-06-01
- Severity: med
- Bump: patch
- Issue: [notTag/Space-Tag-CLI#13](https://github.com/notTag/Space-Tag-CLI/issues/13)

## Repro
When an agent finishes a turn, turn-end.sh resolves the agent's yabai window and fires flash_space with that space id and window id; flash-listener.sh then compares the trigger window to whatever window you currently have focused. If they match — you're sitting on the agent's window — the pill blinks the tool color 5 times and settles back to the focused color. If they don't match — you're in a different app on the same space, or off on another space — it instead paints the pill that tool color solid and drops a marker file (<tool> <win>) so space.sh keeps re-applying the color on every repaint instead of letting a focus change wipe it. That solid hold stays put until flash-reconcile.sh (watching front_app_switched + space_change) sees the agent's trigger window gain focus, at which point it deletes the marker and repaints the pill back to its normal steady color.

## Expected
If an application that triggered a flash event or a solid-hold event closes, the pill color should clear back to default.

## Actual
When the triggering application closes, the pill color does not change back to default — it stays the tool color (stale marker never reconciled because reconcile only fires on the trigger window *gaining focus*, which a closed window never does).
